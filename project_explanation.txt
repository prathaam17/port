Project Complete End-to-End Explanation: NMPA Port Inventory Management System (PIMS)

=========================================
1. COMPLETE PROJECT OVERVIEW
=========================================
This project is an enterprise-grade, role-based terminal logistics application designed to simulate operations at the New Mangalore Port Authority (NMPA). The software manages ship cargo unloading, warehouse storage bay allocations, customs document audits, gate pass control (weighbridge operations), invoicing, and audit logging. It provides customized dashboards for seven port roles (Super Admin, Port Operations, Warehouse Manager, Customs Officer, Gate Officer, Finance, and Shipping Agent) to ensure smooth cargo lifecycles.

=========================================
2. PURPOSE OF THE PROJECT
=========================================
The key objectives of this system are to:
- Digitalize manual port operations, replacing paper cargo manifests and gate logs.
- Enforce strict regulatory compliance by blocking cargo dispatch until both Customs Clearance and Billing Invoices are approved.
- Automate storage fee calculations, including demurrage penalties for cargo staying longer than the 3-day free tier limit.
- Provide a single source of truth for tracking active port inventory and dock capacities.
- Log all operator actions into security audit logs for administrative oversight.

=========================================
3. TECH STACK USED & WHY
=========================================
FRONTEND:
- React.js: Builds the frontend application as a Single Page Application (SPA). This results in lightning-fast component updates without full-page reloads, mimicking desktop terminal software.
- Tailwind CSS: Enforces a structured, modern layout using utility classes, allowing for fully responsive sidebars, data grids, and modals.
- React Router DOM: Handles secure client-side navigation between different staff dashboards.
- Context API: Manages global states, such as storing session tokens and verifying user roles across the application.
- Axios: Sends JSON requests to the backend server with automatic JWT authorization header injection.

BACKEND:
- Node.js: Handles the server-side runtime using JavaScript, unifying the development stack.
- Express.js: Simplifies routing, request parsing, and error-handling middlewares for high-throughput port logistics APIs.
- JSON Web Token (JWT): Encodes session data securely, allowing the backend to remain stateless while verifying authorization.
- Bcrypt.js: Hashes passwords before saving them, preventing plaintext credential exposure in the database.

DATABASE:
- PostgreSQL: An advanced, open-source object-relational database. Chosen for its strict ACID compliance, transactional integrity, and strong support for complex joins and relationships.
- Sequelize ORM: Object-Relational Mapper that maps JavaScript objects to PostgreSQL tables. It automatically generates SQL queries, handles associations, prevents SQL injections, and simplifies migrations.

=========================================
4. FRONTEND ARCHITECTURE & FOLDER STRUCTURE
=========================================
Location: `frontend/`
- `src/components/`: Reusable interface widgets (Navbars, Sidebar navigations, QR code displays, Modal views).
- `src/pages/`: Modular page views loaded conditionally:
  - `auth/`: Login page with quick-login autofill testing cards.
  - `Dashboard.jsx`: Shows customized metrics depending on who logs in.
  - `CustomsClearance.jsx`: Table-view for auditing and approving cargo documents.
  - `UserManagement.jsx`: Administrative dashboard for managing staff and logs.
- `src/context/`: Context declarations like `AuthContext.js` and `NotificationContext.js`.
- `src/services/api.js`: Configuration for Axios, inserting `Authorization: Bearer <token>` automatically.
- `src/App.jsx`: Global router assigning routes to specific dashboards using Protection Guards.

=========================================
5. BACKEND ARCHITECTURE & API STRUCTURE
=========================================
Location: `backend/`
The backend follows the Controller-Model-Route layout:
- `models/`: Contains the Sequelize schemas defining PostgreSQL tables and foreign key rules.
- `controllers/`: Contains the business logic. (e.g. `cargoController.js` logic for uploading manifests, unloading cargo, allocating storage).
- `routes/`: Express endpoints connecting incoming HTTP requests to controllers.
- `middleware/`: Houses `auth.js` for JWT verification and role permissions checks.
- `config/db.js`: Initializes Sequelize and connects to PostgreSQL.
- `server.js`: The application root that mounts the middleware, routes, connects to the database, and boots up the HTTP server.

=========================================
6. DATABASE SCHEMA & RELATIONSHIPS
=========================================
The database contains 13 tables managed via Sequelize:
- `Roles` & `Users`: Users belong to a Role (`roleId` foreign key).
- `Warehouses` & `YardLocations`: A Warehouse has many YardLocations (`warehouseId` foreign key).
- `Cargo`: Refers to a `Warehouse` where it is stored (`warehouseId` foreign key).
- `CustomsClearance`: Belongs to a Cargo (`cargoId` foreign key) and has an inspecting Customs Officer (`officerId` foreign key).
- `GatePass`: Refers to a `Cargo` and a `Truck` (`cargoId` and `truckId` foreign keys).
- `Invoices` & `Payments`: An Invoice is generated for a `Cargo` and can have many `Payments`.
- `AuditLogs`: Records username, role, action, and details.

=========================================
7. AUTHENTICATION & AUTHORIZATION FLOW
=========================================
1. Client logs in via `POST /api/auth/login`.
2. Backend checks email, uses Bcrypt to verify password, and generates a JWT token containing `userId` and `role`.
3. Frontend saves this token in `localStorage`.
4. Subsequent requests include the header: `Authorization: Bearer <token>`.
5. Backend `verifyToken` middleware decodes the token.
6. The `checkRole(['Customs Officer', 'Super Admin'])` middleware blocks unauthorized users from hitting specific sensitive routes.

=========================================
8. FULL REQUEST-RESPONSE LIFECYCLE (Example)
=========================================
Scenario: Customs Officer approves clearance for a Cargo consignment.
1. Customs Officer selects "Release Authorized" in the React UI.
2. Axios triggers `PUT /api/customs/:id` with the payload `{ status: 'Approved', remarks: 'Passed' }`.
3. The Express routing picks up the request in `routes/customs.js`.
4. JWT token is parsed and verified by the auth middleware.
5. The request hits `customsController.updateCustomsStatus`.
6. Sequelize updates the `CustomsClearances` table row.
7. The controller automatically marks the associated `Cargo` status as 'Customs Approved'.
8. The controller calculates demurrage fees (using `cargo.entryTime`) and inserts a new Invoice record in the `Invoices` table.
9. DB sends success. Express responds with `200 OK` and invoice info.
10. React receives response, triggers success toast, and refreshes the data tables.

=========================================
9. DATA FLOW: FRONTEND -> BACKEND -> DATABASE
=========================================
Frontend Event 
 -> API Request with JWT Header 
 -> Route Handling 
 -> Middleware Role Auditing 
 -> Controller SQL Generation 
 -> Sequelize ORM 
 -> PostgreSQL Query Execution 
 -> DB Transaction Commit 
 -> JSON Response payload 
 -> Axios Resolve 
 -> State Update 
 -> Component Redraw.

=========================================
10. FEATURE-WISE RESPONSIBILITY
=========================================
- `backend/controllers/authController.js`: Controls login/auth profiles.
- `backend/controllers/cargoController.js`: Controls cargo unloading and warehouse space allocations.
- `backend/controllers/customsController.js`: Controls document audits and automatic invoice generation.
- `backend/controllers/gateController.js`: Controls weighbridge scans, gate entry, and dispatch inventory deductions.
- `backend/controllers/billingController.js`: Handles payment settlements.

=========================================
11. STATE MANAGEMENT FLOW
=========================================
- Global State: `AuthContext.js` provides details about the current logged-in user and role to the entire frontend.
- Local State: Components query local data and save them in state hooks (e.g. `requests` list in `CustomsClearance.jsx`).

=========================================
12. ERROR HANDLING FLOW
=========================================
- Backend: Wrapped in `try/catch`. Database or validation errors return a `500` or `400` status code with a descriptive JSON message.
- Frontend: Intercepts error responses and uses `NotificationContext` to display alert toast banners to the user, ensuring the app does not crash.
