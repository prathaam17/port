The NMPA Port Logistics application is integrated with a comprehensive and secure **REST API Backend** structured into modular resource routes. Below is the directory of all integrated APIs grouped by operational domain:

### 1. Authentication & Security (`/api/auth`)
*   `POST /api/auth/login` - Authenticates user credentials, generates a secure JWT token, and assigns a role.
*   `GET /api/auth/me` - Retrieves information about the currently authenticated session user.

### 2. User Directory & RBAC (`/api/users`)
*   `GET /api/users` - Lists all port operators registered in the system (Super Admin only).
*   `POST /api/users` - Creates a new operator account (Super Admin only).
*   `PUT /api/users/:id` - Updates role privileges and account statuses for operators (Super Admin only).
*   `DELETE /api/users/:id` - Revokes operator credentials (Super Admin only).
*   `GET /api/users/roles` - Lists the 7 official NMPA security role tiers available.

### 3. Cargo Manifest & Dock Operations (`/api/cargo`)
*   `GET /api/cargo` - Returns lists of all registered cargo shipments.
*   `GET /api/cargo/:id` - Retrieves comprehensive timeline details for a single cargo.
*   `POST /api/cargo/manifest` - Uploads manifest Excel files or Bill of Lading attachments (Shipping Agent / Super Admin only).
*   `PUT /api/cargo/:id/unload` - Logs vessel unloading statuses and cargo dimensions (Port Operations Officer only).
*   `PUT /api/cargo/:id/allocate` - Assigns transit sheds, warehouse slots, or berths (Warehouse Manager only).

### 4. Customs Clearance Module (`/api/customs`)
*   `GET /api/customs` - Lists pending, approved, and rejected customs declarations (Customs Officer only).
*   `PUT /api/customs/:id` - Processes clearances, updates status, and logs inspector details (Customs Officer only).

### 5. Warehouse & Damage Management (`/api/warehouse`)
*   `GET /api/warehouse` - Retrieves occupancy charts, space limits, and stock maps for Transit Sheds and Godowns.
*   `GET /api/warehouse/:id` - Returns lists of cargo currently tracked inside a specific warehouse.
*   `POST /api/warehouse/damage` - Submits a surveyor cargo damage report (Warehouse Manager only).
*   `GET /api/warehouse/damaged/reports` - Lists historical damage logs and cargo surveys.

### 6. Gate Security Passes (`/api/gate`)
*   `GET /api/gate` - Retrieves historically logged gate entry/exit passes.
*   `POST /api/gate/pass` - Generates a new gate entry pass for trucks (Gate Officer / Shipping Agent).
*   `POST /api/gate/entry` - Registers truck scale weights and timestamps on entry (Gate Officer only).
*   `POST /api/gate/exit` - Processes cargo exit authorization and exit timestamps (Gate Officer only).

### 7. Financial Billing & Invoicing (`/api/billing`)
*   `GET /api/billing` - Lists all generated harbor invoices and port storage fees.
*   `GET /api/billing/payments` - Retrieves historic transactional payments.
*   `PUT /api/billing/:id/pay` - Receives and processes payments, shifting invoice state to "Paid" (Finance officer only).

### 8. Dashboard Analytics & Monitoring (`/api/reports`)
*   `GET /api/reports/dashboard` - Computes live KPI counts (vessels, active cargo, warehouse utilization, revenue).
*   `GET /api/reports/analytics` - Provides transactional charts, monthly trends, and volumetric cargo reports.
*   `GET /api/reports/notifications` - Pulls system alerts broadcasted for specific user role groups.
*   `PUT /api/reports/notifications/read` - Marks active notifications as read.
*   `GET /api/reports/audit-logs` - Displays structural system logs of historical operator activities (Super Admin only).
