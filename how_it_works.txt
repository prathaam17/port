Port Inventory Management System (PIMS) - Step-by-Step Workflow

1. SHIPPING AGENT MODULE (Initiation)
- Task: The Shipping Agent starts the process by uploading a Cargo Manifest for incoming cargo.
- Action: Enters cargo details (type, weight, consignee).
- Outcome: Cargo is registered in the system with "Pending" customs status.

2. CUSTOMS OFFICER MODULE (Clearance)
- Task: The Customs Officer reviews the pending cargo manifest.
- Action: Inspects the documents and approves or holds the cargo.
- Outcome: Once approved, the cargo status changes to "Cleared".

3. PORT OPERATIONS OFFICER MODULE (Unloading)
- Task: Monitors the ship berthing and cargo unloading.
- Action: Logs the unloading task and changes the cargo status from "Cleared" to "Unloaded".
- Outcome: The cargo is now physically on the port and awaits warehouse allocation.

4. WAREHOUSE MANAGER MODULE (Storage Allocation)
- Task: Allocates space for the unloaded cargo in the yard or warehouse.
- Action: Selects a warehouse, assigns a specific yard location, and updates the cargo status to "In Storage".
- Outcome: Cargo is safely stored. The system starts tracking storage days for billing.

5. FINANCE OFFICE MODULE (Billing & Invoicing)
- Task: Calculates storage fees and demurrage (penalties for overstaying).
- Action: Issues invoices for the stored cargo. Can also process payments.
- Outcome: An invoice is generated. The shipping agent or consignee must settle this invoice before the cargo leaves.

6. GATE OFFICER MODULE (Dispatch & Exit)
- Task: Manages the exit of trucks carrying the cargo out of the port.
- Action: Verifies that customs are cleared, invoices are paid, and the gate pass is valid. Scans the QR code to dispatch the cargo.
- Outcome: The cargo is marked as "Dispatched" and leaves the port inventory. The process is complete.

Summary of Flow:
Shipping Agent (Upload) -> Customs Officer (Approve) -> Port Operations (Unload) -> Warehouse Manager (Store) -> Finance (Bill) -> Gate Officer (Dispatch)
